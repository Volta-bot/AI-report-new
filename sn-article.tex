\abstract{
Traffic prediction is an integral part of Intelligent Transportation Systems (ITS) amid growing urbanization needs. Even though deep learning methods usually give highly accurate predictions, traditional centralized prediction models come with several disadvantages such as privacy risks, high-bandwidth requirements, and processing latency. To address these problems, this thesis proposes a new privacy-preserving edge-based traffic prediction framework using Federated Learning (FL). Specifically, a customized Multi-Horizon Long Short-Term Memory (LSTM) model is deployed at the edge with the help of Federated Learning, where only the parameters are transferred to the centralized server via Federated Averaging (FedAvg) algorithm. The framework further implements Louvain clustering and graph partitioning methods to select edges and overcome spatial disparities, respectively. SUMO simulation experiments and two real-world benchmark datasets (PeMs-Bay, METR-LA) are utilized to validate the proposed framework's performance. The experiments show that the privacy-preserving federated traffic prediction model achieves 92-96\% accuracy of the baseline model but saves up to 80-262 times more communication bandwidth.
}

\subsection{Experiment Setup}

\subsubsection{Simulation Environment}

The experiment is performed using the following software stack: Traffic simulation via SUMO (Simulation of Urban Mobility) v1.18.0, Federated Learning via Flower Framework v1.5.0, deep learning via PyTorch v2.0.1, and a Streamlit-based monitoring dashboard.

The following pseudo-code outlines the three core loops of the system: the SUMO simulation loop, the edge client loop, and the federated learning server loop.

\begin{algorithm}[H]
\caption{SUMO Simulation Loop}
\begin{algorithmic}[1]
\State Initialize SUMO simulation with network, route, and config files
\State Open WebSocket server
\While{simulation\_step $<$ MAX\_STEPS}
    \State Execute one simulation step
    \State Extract traffic features (speed, count, density) per region
    \For{each subscribed edge client}
        \State Send traffic data via WebSocket
    \EndFor
    \State Advance to next timestep
\EndWhile
\State Shut down simulation and close WebSocket server
\end{algorithmic}
\end{algorithm}

\begin{algorithm}[H]
\caption{Edge Client Loop}
\begin{algorithmic}[1]
\State Initialize edge client with assigned region IDs
\State Connect to SUMO server and subscribe to assigned regions
\While{running}
    \If{data received from SUMO}
        \State Parse message and append to local buffer
    \EndIf
    \If{enough data to train}
        \State Sample training data from buffer
        \For{each epoch}
            \State Train local model on sampled data
        \EndFor
        \State Update local model weights
    \EndIf
    \If{FL round triggered}
        \State Receive global parameters from FL server
        \State Update local model with global parameters
        \State Train local model for FL round
        \State Send local update to FL server
        \State Evaluate on local data and send metrics to FL server
    \EndIf
\EndWhile
\end{algorithmic}
\end{algorithm}

\begin{algorithm}[H]
\caption{Federated Learning Server Loop}
\begin{algorithmic}[1]
\State Initialize global model and bind to server address
\For{each round $r = 1$ \textbf{to} MAX\_ROUNDS}
    \State Select a fraction of available edge clients
    \For{each selected client}
        \State Send current global model parameters
    \EndFor
    \For{each selected client}
        \State Receive local model update
    \EndFor
    \State Aggregate all updates using FedAvg:
    \[
        w_{\text{global}} \leftarrow \sum_{k=1}^{K} \frac{n_k}{n}\, w_k
    \]
    \State Update global model with aggregated parameters
    \State Collect and aggregate validation metrics from clients
    \State Log round metrics to disk
\EndFor
\end{algorithmic}
\end{algorithm}

\subsubsection{Dataset Streaming via TraCI}

Data is streamed from the SUMO simulation using the TraCI interface. SUMO runs as a server that enables real-time interaction through TCP, executing simulation steps at regular intervals (typically every second). During each step, the system extracts data such as traffic metrics at the edge level (e.g., vehicle count, speed), junction level (e.g., traffic light status, waiting vehicles), and individual vehicle level (e.g., position, speed, trip progress).

\begin{verbatim}
1. Connect to SUMO server via TraCI interface
2. Subscribe to edges and junctions of interest
3. For each simulation step:
   a) Advance SUMO by one timestep
   b) Query subscribed objects for metrics
   c) Aggregate metrics by region/client
   d) Publish via WebSocket to all connected edge clients
   e) Store metrics for offline analysis
\end{verbatim}

\subsection{Evaluation Metrics}

Model performance is evaluated using three standard metrics: Mean Absolute Error (MAE), Root Mean Squared Error (RMSE), and Mean Absolute Percentage Error (MAPE), defined as follows:

\begin{equation}
  \mathrm{MAE} = \frac{1}{n}\sum_{i=1}^{n}\left|\hat{y}_i - y_i\right|
\end{equation}

\begin{equation}
  \mathrm{RMSE} = \sqrt{\frac{1}{n}\sum_{i=1}^{n}\left(\hat{y}_i - y_i\right)^2}
\end{equation}

\begin{equation}
  \mathrm{MAPE} = \frac{1}{n}\sum_{i=1}^{n}\left|\frac{\hat{y}_i - y_i}{y_i}\right| \times 100\%
\end{equation}

where $\hat{y}_i$ is the predicted value and $y_i$ is the ground truth value at time step $i$.

\subsection{Experimental Results on Benchmark Datasets}

The core of the empirical study focuses on validating the Federated Learning (FL) framework using the Multi-Horizon LSTM model across two distinct urban environments: PeMS-BAY (California highway) and METR-LA (Los Angeles city streets). The results are categorized into convergence stability, temporal forecasting accuracy, and a comparative trade-off analysis.

\subsubsection{Convergence Analysis}

The efficiency of the Federated Averaging (FedAvg) algorithm was monitored over 50 communication rounds to ensure that distributed training can match the stability of centralized methods.

\begin{figure}[H]
  \centering
  \includegraphics[width=0.8\textwidth]{Figure/pems-bay-fl_convergence_rounds.png}
  \caption{Loss function reduction over 50 communication rounds.}\label{fig:convergence}
\end{figure}

\textbf{Rapid Global Learning:} During the first 10 rounds, the global model experienced a ``warm-up'' phase where the loss function decreased sharply. For PeMS-BAY, the error dropped by 41.3\%, while METR-LA saw a 43.9\% reduction. This indicates that the model quickly identifies fundamental traffic periodicities (e.g., morning vs.\ evening peaks) shared across all geographic zones.

\textbf{Local Feature Integration:} From rounds 11 to 30, the learning rate stabilized as the model began to reconcile non-IID (non-independent and identically distributed) data from different edge clients.

\textbf{Final Convergence:} By the 50th round, the system achieved a total error improvement of 62.5\% (PeMS-BAY) and 68.4\% (METR-LA). The higher improvement in METR-LA suggests that federated learning is particularly robust at extracting meaningful patterns from highly complex urban data.

\subsubsection{Multi-Horizon Temporal Forecasting}

To test the model's utility for real-time traffic management, predictions were evaluated at three critical look-ahead horizons. Table~\ref{tab:horizons} summarizes the results.

\begin{table}[h]
\caption{Multi-Horizon Forecasting Performance (MAPE \%)}\label{tab:horizons}
\begin{tabular}{@{}lccc@{}}
\toprule
\textbf{Dataset} & \textbf{15 min} & \textbf{30 min} & \textbf{60 min} \\
\midrule
PeMS-BAY (Highway) & 3.88\% & 5.42\%* & 7.80\% \\

\bottomrule

\textbf{Short-term (15 min):} Both datasets showed exceptional accuracy. PeMS-BAY maintained a MAPE of 3.88\%, demonstrating that highway speeds are highly predictable in the immediate future.

\textbf{Mid-to-Long-term (30--60 min):} As the horizon expanded to one hour, the MAPE increased to 7.80\% for PeMS-BAY and 11.02\% for METR-LA.

\textbf{Reliability:} Despite the increased difficulty of long-term forecasting, all MAPE values remained significantly below the 12\% threshold. This confirms that the Multi-Horizon LSTM architecture successfully captures long-range temporal dependencies without suffering from vanishing gradients, a common issue in standard RNNs.

\textbf{Dataset Comparative Analysis:} PeMS-BAY (freeway) consistently outperformed METR-LA (urban) by approximately 35--40\% in MAPE. This disparity is attributed to the inherent volatility of stop-and-go urban traffic versus the more stable flow of highway networks.



The error growth across horizons is sub-linear, demonstrating the robustness of the Multi-Horizon LSTM architecture in capturing long-term temporal dependencies.

\subsubsection{Geographic Fairness and Client-Level Analysis}

A critical concern in Federated Learning is whether the global model performs equitably across all participants, especially under non-IID data conditions.



\textbf{Highway Consistency (PeMS-BAY):} Performance across the 6 edge clients was remarkably uniform. At the 15-minute horizon, the Coefficient of Variation (CV) was a mere 2.8\%, with a narrow performance spread of only 0.17 km/h. This validates that graph-based partitioning successfully creates regions with coherent traffic dynamics.

\textbf{Urban Heterogeneity (METR-LA):} In contrast, the urban network exhibited 3$\times$ higher variance compared to the highway scenario. Client-0 (Downtown LA) achieved the highest accuracy (MAE 2.31 at 15 min) due to highly regulated commuting cycles. Conversely, Client-2 (Port Area) faced the highest error rates (MAE 3.15) driven by irregular heavy-vehicle movements and event-driven congestion.

\textbf{Collaborative Boost via Knowledge Transfer:} Clients in irregular zones derived significant benefits from the federated framework. By effectively borrowing structural traffic logic from stable downtown nodes, the Port Area client stabilized its MAPE at 8.91\%, whereas a local-only training approach would likely have resulted in errors exceeding 12\%.

\textbf{Scalability and Fairness:} The results confirm that even in inherently difficult regions, clients can achieve reasonable accuracy through federated knowledge access without sacrificing data sovereignty.

\subsubsection{The Privacy--Performance Trade-off}

To validate the effectiveness of the proposed framework, the FL model was benchmarked against a centralized baseline (the ``Oracle'' scenario), where all historical traffic data is aggregated on a single server for training.



\textbf{Accuracy Preservation:} The FL model successfully retained approximately 91.5\%--92\% of the accuracy achieved by the centralized counterpart. The minor absolute increase in MAPE (approximately 0.6\%--0.92\%) is categorized as a negligible privacy tax --- a small cost for the substantial security gains provided.

\textbf{Root Causes of the Performance Gap:}
\begin{itemize}
    \item \textit{Communication Constraints:} Unlike centralized training, which allows for continuous gradient updates with full precision, FedAvg relies on discretized aggregation at fixed communication rounds.
    \item \textit{Data Non-IID Effects:} Statistical heterogeneity across edge nodes (e.g., complex traffic patterns in port areas) creates a residual disadvantage that global averaging cannot fully eliminate.
\end{itemize}

\textbf{Systemic Efficiency and Regulatory Compliance:}
\begin{itemize}
    \item \textit{Data Privacy:} Raw sensor data never leaves the origin point, ensuring that granular mobility patterns remain local. This privacy-by-design architecture inherently satisfies strict global regulations such as GDPR and CCPA.
    \item \textit{Bandwidth Optimization:} The framework achieved a transformative 262$\times$ reduction in communication overhead. Instead of streaming massive high-frequency raw datasets, the system only transmits compressed model parameters (weights) every round.
    \item \textit{Edge Readiness and Latency:} With an optimized model size of only 20 KB, the architecture is fully compatible with resource-constrained edge devices. This enables real-time local inference with latencies below 100 ms, facilitating immediate proactive traffic management that centralized systems (burdened by round-trip query times) cannot match.
\end{itemize}

\textbf{Operational Conclusion:} For Intelligent Transportation Systems, where signal control algorithms typically tolerate a $\pm$5--10\% error margin, the FL model's performance is functionally equivalent to centralized systems while offering superior operational resilience and security.
